Evolution of LASI 7

7.0.95

  • The toolbar Pan buttons  Lft, Up, Dn  Rgt  have been changed to move the screen about 10% or 50% if TAB is pressed. 

  • If keyboard arrow keys are pressed the screen moves about 10% or 50% if TAB is pressed.

  • The cursor toggles between a normal cross and the LINE cursor if the "L" key is pressed.

  • A toggle button has been added for the "L" key.

  • Many minor changes to the Help files.

7.0.94

  • LasiDrc the design rule  checker has been revised to have a more intuitive setup procedure. It should be still be compatible with older DRC files.

  • Help files have been updated to better explain parameters Distance and Resolution in DRC files.

  • Some DEC errors were found in the Imager Addon. These are fixed.

7.0.93

  • Tooltips have been turned off since they often cause unnecessary redraws on some versions of Windows. Pressing F1 and clicking a command button reads a more complete Help item for that button.

  • Minor internal changes to  the main editor and most utility programs.

  • Addon layouts have been recompiled so that any recent changes will be included.

7.0.92

  • Tooltips can now be hidden or shown  using the Cnfg  command by checking or unchecking "Hide All Tooltips.". This is applied immediately without having to restart LASI.. Tooltips can cause annoying redraws on some versions of Windows.

  • Tooltip timing has been adjusted slightly.

  • Minor changes to utility programs.

  • Several changes to Help files and web pages .

7.0.91

  • Major and Minor changes to the main editor Lasi7.exe. 

    • Pressing ESC and not SPACE now  causes  a Quick Draw. This is more intuitive and lets SPACE do only measurements.

    • Button Tooltips have been removed in the LASI editor and in most utility programs. On some versions of Windows a tooltip popup 
      caused a redraw of the main drawing,  Tooltips are not  very useful since LASI has a more elaborate Help system .

    • The position of color and fill samples under the Cnfg command editors have been corrected.

    • An old bug in the Show command in System  Mode has been fixed.

    • The Import and Export buttons have been removed from System Mode.

    • The MakeTFF utility now works correctly in batch mode.

    • Some reprogramming of System Mode commands.

    • Some changes to LVS feature in LasiCkt utility.

    • Improved Layer Table display for different version of Windows and Linux.

  • As usual some minor changes to the Help files.

7.0.90

  • An old bug has been fixed in the Tput command.. Sides of boxes that were active now remain active.

  • Rot and Flp commands only work on boxes with all sides active and paths with all vertices active. 

7.0.89

  • An Aspect Adjust option has been added to the configuration Cnfg command that lets the Lasi7 editor change
    the X to Y ratio of a drawing. This is to compensate for screen resolutions that may not be in the common 3:4
    or 9:16 ratios as with some laptop or tablet screens, or if you use custom resolution options.

  • When running on Windows, line dash types have been slightly modified to be better spaced and look better on hardcopies
    Running on Linux they default to the normal Windows style or solid lines.

  • Some changes to the way the screen redraws when using the Ctrl key.

  • The screen now redraws with a white background if Alt+D is pressed. This is for using the Print Screen key.

  • When using the Print command the image is not automatically rotated to better fit the paper. The Rotate setup option must now be used.

  • There is now a finer cross hatch fill pattern available if Fine XH Fill is checked under the Attr command.

  • Grids are now preserved when the screen is redrawn and must be toggled on and off.

  • The "Drid=" indicator at the bottom left of the screen turns off if the Grid button is off.

  • Some minor changes to Help files

7.0.88

  • The Edulib Cell Library and the Basic Tutorial have been revised to include Top.gds and Opamp.gds files examples of GDS conversion.

  • The GDS Record Text file written or read by the Tlc2Gds or Gds2Tlc utilities respectively now add "[NUL]" to a string to indicate
     that the text string was extended to an even length of bytes by a null ASCII 0 character. Even byte strings are required by GDS format.

  • The Imager tutorial now contains a Spice CIR file Imager_a_l1.cir compiled from the cell Imager_a_l1 by the LasiCkt utility.
    See the Imager tutorial layout and Help for more information.

  • Several changes to various Help files.

7.0.87

  • Layers are now automatically limited to 1-256 by the Cnfg command Layer Editor.

  • Unused  layers are now initialized to 0 and named "NONE" in the Lasi7  Layer Table.

  • Unused Layer Table  check boxes are grayed-out in the  Lasi7  editor and several utility programs.

  • The Otatutor2 tutorial has been revised to include a "flat" version of the layout.

  • A possible compile error that caused a period  to be missing between a  device name 
    and a connector number in a Node List File made by LasiCkt version 7.0.86 should be fixed.

  • Right and Left check boxes for Connector Ends in the LasiMx utility setup were reversed and should be fixed.

  • The Gds2Tlc converter utility was always writing TLC files in the old style. This should be fixed.

  • Several  Addon tutorials have been updated and have had their Help files recompiled.

7.0.86

  • Improvements to the Print  (or hard copy) command.

  • The text reference mark no longer prints on hard copies.

  • The cell origin mark only prints on hard copies if shown on the screen (R-key or toggle).

  • More elaborate  Print command Help.

  • The text font file Font5.tff has been added to the standard Font command list.

  • The Cmd  command "Safe Mode" has been replaced with :"Build Mode" that will only add objects to a drawing.

  • The Cmd command now has a "Step" button that  will only show and execute one script command line at a time.

  • If a script  command is being run, but fails, it is cleared and Aput  is executed and Get becomes the resident command.

  • If a script is being run, but fails, the original drawing can be recovered by a simple Undo. 

  • The Polar Cursor (P-key or toggle) can be turned on and off during the execution of a cursor input command. 

  • When a cell is loaded the Polar Cursor is turned off and the command is reset to Get.

  • Some changes to pop-up text message in all programs.

7.0.85

  • A check for missing TLC cell files has been added at Lasi7 startup.

  • The Remove command now only deletes the cell name from the Cell Collection list and keeps the TLC file,

  • When a cell is loaded a Load Backup file is saved that can be retrieved if you just want to reload the original cell. or even if Windows crashes. See Undo 

  • The "S" toggle button has been removed. Now pressing CTRL causes Draw to draw small objects and lower cell text reference points.

  • When executing a Script File with the Cmd command each command now is stepped and displayed one command at a time. 

  • Better checks are now made in program setups for illegal characters in cell names and file names.

  • The Spice compiler program LasiCkt has had several internal changes. See LasiCkt Help for more details.

  • The Lasipgl7.exe external program has been renamed to Lasihpgl7.exe since it writes only HP-GL code.

  • A LasiHpgl utility button that runs Lasihpgl.exe has been added to the System Mode menu.

  • The Maketff text font file editor has been moved to a menu button in System Mode.

  • The text characters drawings now install as a standard drawing "Chars'".

  • A new text font file Font4.tff has been included that is similar to block Font3 but has acute angles in its characters. removed.

  • Browse buttons in most programs now have an "All Files" viewing choice.

  • As usual, minor changes to programs and Help files.

7.0.84

  • Timing has been added to the GDS, DXF and CIF converter programs that better shows the operations during the conversion process.

  • The Design Rule Check utility LasiDrc now has a  message box that lets you abort or continue if the number of error map files exceed a limit.

  • The LasiDrc program now better remembers setup parameters so that if a DRC is aborted it can be rerun and continue where it left off.. 

  • The LasiDrc program has had a possible DRC file PAUSE operator bug fixed that caused error maps to be shown too quickly. 

  • The GDS converter Tlc2Gds has had a bug fixed that could have caused the Write Gds Record Text File setup option to be ignored.

  • The LasiMx router program now checks for identical original and connected cell names so that the original cell cannot be erased.

  • When the configured to run on Linux with Wine the notice "Wine Mode is On" now appears in utility programs when Ctrl-A is typed.

  • When the configured to run on Linux with Wine the standard Windows message box is used in all programs.

  • As usual, some typos and minor bugs have been fixed and Help files have been updated.

7.0.83

  • The "T" key or toggle button now hides or shows text. This should make a drawing a little more readable. 

    The action of the "T" key is now reversed. Your drawings may first come up with text hidden. Press "T" to view.

  • Blocking for certain commands, Del for example, on objects that are not currently visible has been improved.

  • Blocking does not affect objects added or edited from command lines or script files using the Cmd command.

  • LasiCkt the Spice compiler now places Real Node Auto Connectors exactly on the Real Node Text. See the LasiCkt help.

  • Some changes to LasiCkt error detection and messages.

  • Addons have been updated.

  • Help changes and a few small improvements and bugs fixed.

7.0.82

  • LasiCkt the options "Write Tracking Text Script" and "Show Count Messages" are reversed in the Setup. This is fixed.

  • More additions and updates to the Help files.

7.0.81

  • A "Tracking Text" feature has been added to LasiCkt the Spice compiler. See Tracking Text.

  • LasiCkt Connector Text can now contain a comment number that indicates which conductor layer can connect. See Connector Text.

  • Improved LasiCkt setup options and error checks including a Default button for best options. 

  • The SHIFT key now changes commands Get, Fget, Tget, Cget, Outl to their inverse Put, Fput, Tput, Cput, Full and conversely.

  • The CTRL key now does most all other command changes and modifications in the LASI drawing editor.

  • The "M" key now toggles the command menu. See Key Assignment.

  • Many revisions to the Help files.

7.0.80

  • End of year updates, some minor  program changes and rewrites to various Help files.

7.0.79

  • The Spice compiler utility LasiCkt has been revised with better speed and other features and fixes. See LasiCkt Help.

  • Most Addon layouts have been recompiled with LasiCkt using Full Depth and Auto Connectors options and simulated with Spice.

  • An old bug that caused the origin mark not to print and grid dots not to redraw when making hardcopies has been fixed.

  • The editor screen no longer does a automatic Draw command after a hardcopy print.

  • The save and restore window Swin and Rwin commands now have numbered buttons.

  • Many Help files have been "cleaned up" of accumulated changes. 

7.0.78

  • The main editor now sets its toolbar style from the Use Text Toolbar setting under Cnfg.

  • Utility programs now configure their control panels using the text or icon toolbar setting of the main editor.

  • The standard Windows menu bar has been removed for the main editor and the utilities. The menu just duplicated the buttons below it.

  • The LASI editor's toolbar at the top-left of the screen can now be moved to the top-center of the screen. This seems better for widescreen monitors. The toolbar position can be changed by the Center Top Toolbar setting under the Cnfg command. 

  • Standard message boxes that are part of the Windows system can be replaced by "custom" message boxes. If Windows has installed a language other than English the button labels such as "OK" or "Cancel" will be translated into the local language. This inconsistent with other messages which are in English. The message box type can be changed by the Cnfg command. 

  • As usual, some minor changes and corrections have been made to the Help files 

7.0.77

  • Several changes and some bug fixes have been made to the Spice compiler utility LasiCkt. See the LasiCkt Help for specific details.

  • Some editor toggle keys now do an immediate redraw if a redraw is necessary to see the effect of the toggle. This this a handy feature but can be turned off and on with the Cnfg command.

  • When text reference marks are toggled on with the "T" key they normally only appear on the editable drawing level. This reduces drawing clutter from lower cells. The text reference marks on lower cells can still be seen by holding down the CTRL key during a redraw.

  • Vertex marks only show on paths on the top or editable level when toggled on with the "V" key. This also reduces drawing clutter from lower cells.

  • Path center lines when toggled on with the "C" key appear on all levels so you can tell paths from boxes.

  • A parabola generator has been added to the Arc command. Although previously it could be approximated by a section of an ellipse, the new algorithm generates a precise parabola with an exact focal length.

  • The Res command has been revised to work more intuitively. It can more easily be used to measure the lengths of various arcs, polygons or paths.

  • Many small changes and updates to programs and help files.

7.0.76

  • Memory allocations have been increased for the Spice compiler utility LasiCkt for more virtual nodes. This lets the utility add a large number of internal connection nodes to the Spice file to find possible layout errors.

  • LasiCkt can now compile just Connector Text script files and skip the Spice circuit file. The script can then be used to add connectors to a cell and reduce Spice compile time.

  • The the design rule checker LasiDrc and LasiCkt now continuously show the running time. This gives you an indication that the program is running and how long it might take. Both programs can take a long time to run.

  • Most utility program windows can now be minimized and will pop up when finished. In general, if you are careful, long running programs can be minimized to the Windows task bar and can then run other programs.

  • Control panel buttons on utility programs now lock out when a program is running. This prevents multiple instances of the same program or changing setups while a program is running. To stop and unlock a utility before it finishes you need to press ESC.

  • Small changes and fixes have been made to the main editor and most of the utilities.

7.0.75

  • The GDS conversion utilities Gds2Tlc and Tlc2Gds now write text files that translate the binary GDS records to plain text. These files should be educational and very useful as diagnostics by showing what is really written in a GDS file.

  • The Spice compiler LasiCkt has been greatly revised to make it much easier to compile Spice circuit files from layout or schematic. It will run as previous versions, or as a two pass compiler. On the first pass, it writes script files that add Connector Text on real or virtual nodes. On a second pass, it reads the script, temporarily adds connectors to the cells and recompiles. Adding connectors lets you find certain layout errors that might be missed.

  • The Spice compiler LasiCkt has also had some setup changes that eliminate some rarely used options. It also has some improved checks for cell layout errors that can cause incorrect or unworkable Spice circuit files.

  • Most of the LASI Addon tutorials have been updated with revised LasiCkt sample compiles and connector scripts.

7.0.74

  • The progress bar has been removed from the LasiCkt Spice compile display. Running under Windows 7 it was slow and erratic. The bar was not very useful because LasiCkt is an event driven program where it is difficult to accurately measure progress.

  • The rarely used utility programs Lasipgl7.exe and Calma2tlc7.exe now both install in the \Lasi7 folder. Use the Run command to start them.

  • The LASI Setup date should now always be correct. In the 7.0.73 release the date might have been taken from a previous file that had not been deleted.

  • Design Rule Check DRC files for LasiDrc now contain check code sequences for metal overlaps. These checks are currently turned off by comments.

  • On Windows 7 when a tooltip pops up the LASI editor sometimes redraws if you are not using Aero themes. Windows 7 sends the editor a "window paint" message. On large drawings this can be a problem. To prevent this check "Hide Tooltips" under Cnfg. Utility program tooltips in the editor drawing area can also caused a redraw. Hide Tooltips is now also passed to utility programs.

7.0.73

  • The Spice compiler utility LasiCkt can now automatically add connector text to a layout. By adding additional connector text LasiCkt can better detect certain layout errors.

  • The cursor information in the lower left of the editor screen and the editor distance measuring tool now show the cursor vector angle.

  • When calculating parasitic capacitances LasiCkt now reads out the capacitance node names.

  • The Arc command can now generate certain basic spiral types for special integrated devices.

7.0.72

  • Compiling schematics with LasiCkt with Full Depth set did not work correctly in the previous release. The bug should be fixed.

  • An old bug in LasiDrc that caused a white vertical stripe when viewing a PCX bitmap has been fixed.

  • An asterisk "*" can now be used as a comment character in Command Line Script and in DRC files. See the Help.

  • As usual, many small changes and updates in the Help files.

7.0.71

  • You can now save a cell directly in Command Line Script. The Save command has a check box option. This is useful if you want to create and edit individual cells or parts of cell with a text editor and then add them to other cells.

  • Some of the keyboard permanent key assignments in the drawing editor have been changed. See Key Assignment.

  • If a secondary line or rectangle cursor is showing and a key that executes a different command is pressed, the cursor is erased (old bug).

  • When run on Windows 7 the toolbar buttons now show either text or icons, not a combination of both when text only is chosen. 

  • LasiCkt the Spice compiler has been modified to let you use the Full Depth option when compiling schematics. This helps when working with nested schematic drawings.

  • The cell library Edulib has been modified so that when running an LVS with LasiCkt, the layout cells RINGOSC31, DELAYOSC and 4COUNTER now compare correctly with their corresponding schematics. The required changes were:

    Power bus wires with node text Vdd and Gnd (0) were added to the schematics so that the wires overlay the connector text the lower schematic cells. Otherwise the schematics would not compile correctly, reporting an open connector error, and .NOD files for LVS would not be written. 

    Subcircuit device names (X1, X2, etc) have been added to the schematics that correspond to the same layout cells.

    We also changed the former name RINGOSC back to RINGOSC31 to indicate the number of stages.

  • Many small changes and updates in the Help files.

7.0.70

  • The Cmd menu button, now runs LASI Script Files that execute an unlimited number of editing commands. Scripts are text files that consist of literal "command lines" that mimic menu buttons so that commands can be executed without manual mouse input.

    The real purpose of script is to allow applications to virtually construct cells or even complete layouts and then translate to LASI drawings. Scripts can be written with a simple text editor, calculated and written by programming languages such as C/C++, Basic, or Java, or by using popular mathematical modeling and analysis applications such as Mathlab or Mathematica.

    For more details read Command Lines and Script Files.

  • The mouse cursor when switched to polar coordinates can now be set to any angle. Pressing "P" or clicking the P toggle button brings up an angle box. When the mouse is moved the cursor moves radially. The angle is not absolutely exact since points are always in a Cartesian working grid.

  • Comment lines in DRC, LDF and Script files now start with only the semicolon ";" character. Previously an asterisk could also be used for comments, and it could be confusing. This change does not include the asterisk used for comments in Spice files or using the asterisk to indicate a "wildcard".

  • Many small changes and Help file updates.

7.0.69

  • The Import command now creates a text object with the name of a missing cell if the TLC file of the cell is not found. Previously a "false" cell was created with the name of the missing cell. Now only a text marker is added. This prevents creating false cells in a drawing with the same name as the correct cell and possibly importing them into other drawings. The converters Gds2Tlc and Dxf2Tlc work the same way.

  • The layer of the missing cell text object can now be set in the Import pop-up box.

  • The "Zip" button on the Import and Export pop-ups has been removed. It was not very useful since zipping programs have various interfaces. Often the Run command can be used in its place.

  • The "O"  key (Octo toggle) has been replaced by the "P" (Polar toggle) that causes the cursor to snap to 30, 45 and 60deg angles Snapping to 30 and 60deg. should make it easier to draw hexagonal polygons.

  •  When using Print, the Clipboard option is disabled if Modify for Wine on Linux is checked under Cnfg.

  • The Spice compiler LasiCkt now automatically sets Full Depth in the setup when Antenna Rules is enabled.

  • An old bug in the cursor readout in the bottom of the editor screen for some commands has been fixed.

  • The cursor readout now shows incremental x,y distance and absolute distance continuously for certain commands.

  • The installer Lasi70xx.exe now contains an HTML version of the main Help. This can be optionally installed and read by any browser. The installer is a larger download by about 600KB, but installation in some cases (under Linux for example) is much simpler.

  • The CMOS model resource files have been updated and now come in level 8 and level 49 parameter versions.

  • The node to node parasitic capacitance calculation in LasiCkt should now be much faster and should be more accurate.

7.0.68

  • The Gds2Tlc converter has been revised to fix occasional problems when converting GDS files to LASI.

    • GDS Structures and Structure Reference names are now treated as case dependent.

    • Cell hierarchy is now determined by case dependent names, eliminating possible recursive cell nesting.

    • You can now relocate GDS layer 0 to any LASI layer 1-256.

    • The missing cell marker text layer can now be set 1-256.

    • GDS Structure names are always converted to Uppercase.

  • The number of View, Open, Attr command pages has been increased from 4 to 8. Now all 256 drawing layers can be defined.

  • Improvements have been made to the LASI editor.

    • The Dpth command is now simpler and lets you draw only cells with a specific rank.

    • Cells smaller than the Cell Resolution set in Cnfg are shown with alternate pixel dotted outlines.

    • There is a Hide Small Cell Outlines option in Cnfg that lets you omit drawing small cell outlines completely.

    • There are now sixth dash styles.

  • The LasiDrc design rule checker has been improved.

    • It now lets you select bitmap sizes for faster checks on different sized layouts.

    • The Estimated Time readout in the display has been moved to give more space for cell and check info.

  • Utility programs now list the version number on their title bar.

  • Utility programs now write the LASI version number in log and other text files they create.

  • LASI is now compiled using the 2010 Visual C++ library.

  • Various other little tweaks.

7.0.67

  • Several tweaks and fixes have been made to LasiCkt the Spice compiler:

    • The number of remaining nodes while connecting virtual nodesI are now occasionally reported.

    • Only cell objects on layers in the Trace Table are now loaded.

    • Long compiles generally now run about 25% faster.

    • Help has been rewritten with emphasis on using the Full Depth option in the Setup.

  • The distortion of non-orthogonal figures when running on Linux with Wine has been fixed.

  • Print setup on Linux with Wine has been revised to disable some features that do not work.

  • Many changes to Help files.

7.0.66

  • The AutoCAD DXF converter Tlc2Dxf can now optionally convert LASI Boxes to Solid entities. Solids are useful because they are usually solidly filled areas that can be used to make photo masks for PC boards or similar applications.

  • A small converter program has been added to Circuits and Resources that will mostly convert layouts saved in the original Calma format (GDS I, not GDS II) to TLC files. This is for the rare occasion that someone wants to recover ancient layouts.

  • The status window for the Import and Export commands has been moved to the upper left of the screen so not to cover centered pop-up windows.

  • Spice device model resource files have been updated with more recently measured model parameters from MOSIS.

  • Addon tutorials have been updated with newer device models.

7.0.65

  • The Print command has been further revised with options to rotate and change resolution when printing direct. 

  • The Dpth command now has Step Up and Dn buttons that let you look at cell nesting one level at a time.

  • The Spice compiler LasiCkt now considerers a semicolon in layout text as the beginning of a comment and removes the comment from the text string.

  • LasiCkt now uses an inline comment in layout text to attach a node name to a single layout layer. See the LasiCkt Node Text Help topic.

7.0.64

  • The buttons of all programs now adjust for Windows screen resolution DPI settings greater than the normal 96 dpi.

  • The "S" key toggles drawing small objects on and off.

  • The "H" key now toggles drawing of cells on and off.

  • The "M" key now flips between the two edit screen command menus.

  • The Spice compiler LasiCkt can now associate a node name with a single layer. See the LasiCkt Node Text Help topic.

  • The Print command has been updated. See the Print command Help topic

  • LasiDrc has not been erasing old Log files since version 7.0.62. This should be fixed.

  • LasiDrc has not been mapping top boxes and paths if the top cell has no lower cells. This should be fixed.

  • Many other small changes, updates  and tweaks.

7.0.63

  • The Import command now compares each individual cell's TLC header record scale units with the scale units in  the current drawing. If scales are different, you can cancel the import process and change drawing scales before importing, or continue importing cells and resize cells, if necessary. Whichever is easier.

  • Some revisions to the Help, including suggestions on how to change drawing scales and some small program tweaks.

7.0.62

  • The screen copy to a PCX file in LasiDrc (SCPY operator) now runs about 4 times faster. This greatly speeds DRCs when a layout has many errors.

  • Some built-in delays in the LasiFlat program have been reduced or eliminated. Large circuits now run much faster.

  • The LasiFlat program now more accurately removes unnecessary node text when flattening a layout.

7.0.61

  • The node list length in the LasiCkt Spice compiler has been doubled to reduce list overflow errors when compiling antenna rules on large circuits.

  • The algorithm used to step objects in polar coordinates with the Rot command has been improved to avoid accumulated round-off errors.

  • The rectangular stepping command Step can now step rows in a sequential pattern in addition to the original zigzag pattern.

7.0.60

  • LasiCkt can now calculate Antenna Rules using the edge and thickness of patterns being etched in addition to surface area.

  • LasiCkt now has a single setup parameter that rescales both capacitance and antenna rules to microns.

  • A bug in LasiMx that might cause connections to be missed on cells with a large number of connections has been fixed.

  • A bug in LasiCkt that might cause Open Connector errors to be missed on schematics has been fixed.

  • A bug in LasiCkt that might have rarely missed an object in a capacitance calculation has been fixed.

  • A bug in the Cap command that prevented fringe capacitances from being measured manually has been fixed.

7.0.59

  • LasiCkt now searches deeper in cell nesting for node names on conductors. This makes connections between cell levels much more intuitive. See What's New in LasiCkt help.

  • The Startup dialog of version 7.0.58 has been improved. You can now select the cell you want to load or load the last edited cell. 

  • The utilities Dxf2Tlc, Cif2Tlc and Gds2Tlc may have been writing incorrect TLC text records. This is fixed.

  • An old bug in LasiCkt that caused some LVS error logging to be skipped has been fixed.

  • The Organize command now also sorts cells into blocks of the same cell to speed up redraws.

7.0.58

  • When run, the LASI drawing editor Lasi7.exe now checks if the current folder is a Drawing Folder. If not, it pops up a Startup dialog box. The Startup box lets you browse to another folder, or lets you create a new Drawing Folder where you can begin a new drawing..

  • If the LASI system folder and your drawings are installed on a USB flash drive, LASI can run "portable". This should be useful for students or for demonstrations. See Portable Operation in the General Topics Help.

  • When running portable from a flash drive, temporary files can be written to the main hard drive, not the flash drive with a Flash Drive Saver feature,

  • An old bug in LasiCkt has been fixed that could cause an over length subcircuit definition line if the the subcircuit has a large number of connections. 

  • The Depth command now has a "Lowest" depth button that immediately shows you all nested cell levels.

  • Running the Editor now appears as a General Topics Help item. See Running the Editor.

7.0.57

  • Depending on the layout and options the Spice compiler LasiCkt now runs about twice as fast as the previous versions.

  • Checking for open connectors during a LasiCkt Spice compile is now more strict. 

  • There are now option switches in LasiCkt setup to turn off open connector and shorted node checks.

  • There are now cell listing buttons in the LasiCkt Smash and Omit setups.

  • Most tutorials have been upgraded to conform to the more strict open connection rules.

  • Read the LasiCkt help for more.

7.0.56

  • LasiCkt the Spice compiler has been considerably revised and now collects Antenna Rules information.

  • The control panel of LasiCkt has been simplified with different setup categories assigned to separate control buttons.

  • The control panels of most of the utility programs now automatically adjust for title bar size changes.

  • Special single contacts have been added on layer 15 in the CMOS tutorials to help reduce Spice compile times. See Speed Up Tips in LasiCkt help.

  • The "\Lasi\Resource" folder is now used as the common source of certain files referenced in utility program setups. This permits future updates to be done in only one place.

  • As usual many little tweaks.

7.0.55

  • Most programs now check if the current folder is a drawing folder, otherwise they might not work correctly.

  • The Setup program now can create new drawings directly. This avoids the awkward Windows procedure.

  • The size of Spice code lines has been increased to 128 characters to accommodate longer text strings.

  • Ring oscillator cells have been added to the EduLib cell library to check basic process speed.

  • As usual, many small help file changes.

7.0.54

  • The Arc command will now generate closed ellipses and circles directly. Read Arc help.

  • Tutorials no longer have version numbers. The setup window shows the date of the release.

  • Tutorials have been recompiled with XML TLC files and new cell reference file.

  • The default TLC format configuration bug in 7.0.52 should be fixed.

  • The Arc command would sometimes create vertices all at a single point when you were trying to draw a full circle. It was caused by the starting and ending  angles of the arc not being exactly equal. This has been fixed by rounding the floating point angle to 32-bit accuracy.

Some of these involve better optimizing for MEMS type structures, but most simply improve the ability to do design rule checks more easily and accurately. 7.0.2.2

  • The old numerical layer mode in LASI drawing editor has been removed. This was a legacy mode that dated back to the DOS version of LASI. It was basically unnecessary since you can (and should) use the Layer Table. Layer numbers can always be added to the Layer Table using the layer editor under the Cnfg command. Removing this resulted in much more elegant layer procedures.

  • The Layer Table is now translated to a series of layer numbers that appear on the bottom line of the LASI display. You can set this under Cnfg to show the layers Viewed and Opened, or the layers that are Viewer AND Opened (View+Open), the actual layers that can be edited.

  • LasiCkt now has a new check that looks for Node Text on more than one DIFFERENT conductor layers. This reports a warning because the layers may not be connected in a layout. If they is no connection then there will be an ambiguity as to which layer is marked by the Node Text, which can result in incorrect Spice code..

The cells have been redrawn to meet MOSIS SCMOS submicron design rules for processes that do not allow stacked vias (Rule 8.4). The cells are therefore suitable for all but DEEP SCMOS.

A few convenience features that we have been wanting to do for a long time have been added. These include:

  • When you save a cell, LASI can now do an automatic Udup (unduplicate). This is an option under the Cnfg command.

  • When you start LASI the Load or Create a Cell pop-up is always shown. This is no longer an option under the Cnfg command.

  • The Info command now returns information about all active objects.

  • The information at the bottom of the display has been rearranged. This was originally set for a 640x480 screen size. Now with larger screen sizes used almost universally, it makes more sense to interchange the information and layer lines.

  • Also now due to larger screen sizes, there is space at the bottom of the display to show the Text count independently from the Path count.

  • Most of the built-in pauses have been taken out of LasiDrc and LasiCkt. These pauses were put in so that you could read operations as they progressed. With faster new PCs these pauses were actually slowing the programs down. If you want to run a program more slowly, you can now check "Op Pause" in the setup.

  • Top toolbar buttons now fit correctly on a 800x600 screen size.

Previous Changes

In addition to the elimination of internal binary files, there are many other changes, these include:

  • LASI 7 now tries to detect if you have made changes to a cell.

  • If a cell has been changed but not saved, a "(Not Saved)" appears on the title line.

  • If you save a cell from memory to its TLC file, there will be a pop-up that asks if you want to save the cell. Cell management is setup so that you will normally answer "Yes" to this.

  • There is also a check box under the Cnfg command "Disable Ask Before Saving Cell". This causes the above pop-up to be skipped with the default of always saving the cell.

  • There is now a  8 deep stack of Undo files, instead of the previous single Undo file.

  • There is now a Redo button under Undo. This loads the cell as it was just before the last Undo.

  • There is now a Load button under Undo. This loads a cell as it was when it was last loaded into object memory.

  • We have also changed the labeling of the LasiCkt "Compare" feature. This is actually Layout vs. Schematic, and we are now calling it LVS. We have also added browse buttons for node file selection.

  • When viewing an error map with LasiDrc, you can now send a command back to the main drawing to locate the drawing window to the map area.

  • LasiDrc now has a "Checks" button in the Setup which reads the DRC file and lists the check sequences by number and title. This makes it easy to run individual checks.

  • Utility programs now have various checks to prevent them from accidentally overwriting TLC cell files in the drawing folder.

  • The Cnfg command now has a Colors editor. You can now adjust your drawing screen colors by changing the proportion of Red, Green and Blue in the 12 layer color attributes.

  • The number of segments in a single PATH or POLY object can now be 1024.
